Data Private detects and replaces personally identifiable information (PII) in a text with numbered placeholders like `[FIRST_NAME_1]` or `[IBAN_1]`. You send the anonymized text to your LLM. You handle the mapping. Nothing sensitive ever leaves your system.

---

## The problem

Without Data Private, this code sends your user's personal data to OpenAI:

```python
# ❌ Without Data Private — PII sent to OpenAI
response = openai.chat.completions.create(
    model="gpt-4o",
    messages=[{
        "role": "user",
        "content": "Summarize this contract for Jean Dupont, "
                   "IBAN: FR76 3000 6000 0112 3456 7890, "
                   "born 12/03/1985."
    }]
)
```

With Data Private:

```python
# ✅ With Data Private — PII stays in your infrastructure
import requests

# 1. Detect and anonymize
result = requests.post(
    "https://private-data.ai/api/v1/public/extract",
    headers={"X-API-Key": "your_api_key"},
    json={"text": user_message}
).json()

# 2. Send clean text to LLM
response = openai.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": result["text"]}]
)

# 3. Restore values locally — nothing goes back to Data Private
restored = result["text"]
for entity in result["entities"]:
    placeholder = f"[{entity['type'].upper()}_1]"
    restored = restored.replace(placeholder, entity["value"])
```

What OpenAI receives:
```
"Summarize this contract for [FIRST_NAME_1] [LAST_NAME_1], IBAN: [IBAN_1], born [DATE_OF_BIRTH_1]."
```

---

## What Data Private does

- **Detects** 40+ PII types: names, IBANs, emails, phone numbers, addresses, dates of birth, passport numbers, IPs, and more.
- **Anonymizes** by replacing each PII with a typed, numbered placeholder (`[FIRST_NAME_1]`, `[IBAN_1]`, `[EMAIL_1]`…).
- **Returns the entity map** so you can restore original values locally — the mapping never transits through Data Private servers.
- **Supports 6 languages** automatically: English, French, Spanish, Italian, German, Arabic.

---

## What Data Private does NOT do

<Warning>
**Data Private is not a firewall.** If your user asks the LLM to repeat the anonymized text verbatim, the placeholder `[FIRST_NAME_1]` will appear in the response — not the real name, but the token itself will be visible.
</Warning>

- **Does not deanonymize server-side** — the mapping stays in your code, not on Data Private servers.
- **Does not guarantee 100% recall** on out-of-context or highly ambiguous PII (e.g. common first names without surname in a short text).
- **Does not filter intent** — it anonymizes text, not semantics.
- **Does not replace a DPO** — it's a technical tool, not legal GDPR advice.

---

## Guarantees

| Property | Detail |
|---|---|
| **Zero data retention** | Texts submitted to `/extract` are not stored or logged by Data Private |
| **Local mapping** | The entity map (placeholder → original value) is returned in the response and never stored server-side |
| **EU infrastructure** | The API runs exclusively on European servers |
| **Automatic language detection** | No language parameter needed — the model detects it automatically |
