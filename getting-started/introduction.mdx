Data Private détecte et remplace les données personnelles (PII) dans un texte par des tokens réversibles. Tu envoies le texte anonymisé au LLM, tu reçois la réponse, Data Private restaure les valeurs réelles.

Aucune donnée personnelle ne quitte ton infrastructure.

---

## Le problème

Sans Data Private, ce code envoie les données personnelles de ton utilisateur à OpenAI :

```python
# ❌ Sans Data Private — PII envoyées à OpenAI
response = openai.chat.completions.create(
    model="gpt-4o",
    messages=[{
        "role": "user",
        "content": "Résume ce contrat pour Jean Dupont, "
                   "IBAN: FR76 3000 6000 0112 3456 7890, "
                   "né le 12/03/1985."
    }]
)
```

Ce que reçoit OpenAI :
```
"Résume ce contrat pour Jean Dupont, IBAN: FR76 3000 6000 0112 3456 7890, né le 12/03/1985."
```

Avec Data Private :

```python
# ✅ Avec Data Private — PII restent dans ton infra
anon = dp.anonymize(text)

response = openai.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": anon["anonymized_text"]}]
)

result = dp.deanonymize(response.choices[0].message.content, anon["tokens"])
```

Ce que reçoit OpenAI :
```
"Résume ce contrat pour [PERSON_1], IBAN: [IBAN_1], né le [DATE_1]."
```

---

## Ce que Data Private fait

- **Détecte** 60+ types de PII : noms, IBANs, emails, numéros de téléphone, adresses, dates de naissance, numéros de sécurité sociale, passeports, plaques d'immatriculation, coordonnées GPS, et plus.
- **Anonymise** en remplaçant chaque PII par un token unique et typé (`[PERSON_1]`, `[IBAN_1]`, `[EMAIL_2]`…).
- **Réidentifie** en restaurant les valeurs originales dans la réponse du LLM, même si le LLM a réorganisé ou reformulé le texte.
- **Fonctionne en français et en anglais** (autres langues en beta).

---

## Ce que Data Private ne fait PAS

<Warning>
**Data Private n'est pas un pare-feu.** Si ton utilisateur demande au LLM de répéter verbatim le texte anonymisé, le token `[PERSON_1]` apparaîtra dans la réponse — pas le nom réel, mais le token lui-même sera visible.
</Warning>

- **Ne chiffre pas** les données au repos — les tokens sont stockés en mémoire côté client pendant la session, pas sur les serveurs Data Private.
- **Ne garantit pas 100% de recall** sur des PII hors-contexte ou très ambiguës (ex : prénoms communs sans nom de famille dans un texte court).
- **Ne filtre pas l'intention** — il anonymise le texte, pas la sémantique. Si le prompt lui-même est problématique, c'est hors scope.
- **Ne remplace pas un DPO**. C'est un outil technique, pas un avis juridique RGPD.

---

## Garanties

| Propriété | Détail |
|---|---|
| **Zero data retention** | Les textes soumis à `/anonymize` ne sont pas stockés ni loggés côté Data Private |
| **Tokens éphémères** | Les mappings PII → token sont générés à la volée et retournés dans la réponse — ils ne transitent pas par nos serveurs |
| **Infra EU** | L'API tourne exclusivement sur des serveurs hébergés en Europe |
| **Réversibilité garantie** | Un token généré par `/anonymize` est toujours réidentifiable via `/deanonymize` pendant la durée de la session |
