## Data flow

Every call to Data Private follows this pipeline:

```
Raw text (with PII)
        │
        ▼
┌───────────────────────┐
│   /extract endpoint   │  ← 40+ types, 6 languages
└───────────────────────┘
        │
        ▼
  Anonymized text            Entity list
  [FIRST_NAME_1]...    +    [{value, type, score, start, end}]
        │
        ▼
  Your code builds           Your code stores
  the placeholder map   →    the mapping locally
        │
        ▼
  Send anonymized text ────► LLM (OpenAI, Claude, Mistral...)
                                        │
                                        ▼
                              LLM response with placeholders
                                        │
                                        ▼
                         Your code restores original values
                         (string replace — no API call needed)
```

<Note>
  The mapping from placeholder to original value is built by your code from the `entities` array. It never transits through Data Private servers after the initial `/extract` call.
</Note>

## How to deanonymize locally

After calling `/extract`, you have everything you need to restore original values — no second API call required.

```python Python
import requests

result = requests.post(
    "https://private-data.ai/api/v1/public/extract",
    headers={"X-API-Key": "your_api_key"},
    json={"text": "Hello, I'm Jean Dupont and my email is jean@dupont.fr"}
).json()

# Build the local map from entities returned by the API
entity_map = {}
counters = {}
for entity in result["entities"]:
    t = entity["type"].upper()
    counters[t] = counters.get(t, 0) + 1
    placeholder = f"[{t}_{counters[t]}]"
    entity_map[placeholder] = entity["value"]

# result["text"] → "Hello, I'm [FIRST_NAME_1] [LAST_NAME_1] and my email is [EMAIL_1]"

# Send anonymized text to your LLM...
llm_response = "Thank you [FIRST_NAME_1], your message has been sent to [EMAIL_1]."

# Restore locally
restored = llm_response
for placeholder, original in entity_map.items():
    restored = restored.replace(placeholder, original)

print(restored)
# → "Thank you Jean, your message has been sent to jean@dupont.fr."
```

```javascript JavaScript
const response = await fetch("https://private-data.ai/api/v1/public/extract", {
  method: "POST",
  headers: { "X-API-Key": "your_api_key", "Content-Type": "application/json" },
  body: JSON.stringify({ text: "Hello, I'm Jean Dupont and my email is jean@dupont.fr" })
});

const result = await response.json();

// Build local map
const counters = {};
const entityMap = {};
for (const entity of result.entities) {
  const t = entity.type.toUpperCase();
  counters[t] = (counters[t] || 0) + 1;
  entityMap[`[${t}_${counters[t]}]`] = entity.value;
}

// Restore in LLM response
let restored = llmResponse;
for (const [placeholder, original] of Object.entries(entityMap)) {
  restored = restored.replaceAll(placeholder, original);
}
```

## Supported PII types

| Category | Types |
|---|---|
| **Personal Identity** | first_name, last_name, title, date_of_birth, country, city, address, postal_code, national_id_number, identity_card_number, passport_number, passport_expiration_date, driver_license_number, social_security_number, tax_identification_number, cpf, cnpj |
| **Contact** | phone_number, mobile_phone_number, fax_number, email, email_address, social_media_handle, username |
| **Financial** | credit_card_number, credit_card_expiration_date, cvv, bank_account_number, iban, bic, money, transaction_number |
| **Health** | medical_condition, medication, blood_type, healthcare_number |
| **Sensitive** | religion, marital_status |
| **Professional** | organization_name, occupation, registration_number, insurance_company |
| **Vehicles & Travel** | license_plate_number, flight_number, visa_number |
| **Technical** | ip_address, mac_address, password |
| **Dates & Times** | date, time |

<Warning>
  Recall is not 100% on out-of-context PII. A standalone first name in a short text (e.g. "Thanks Pierre") may not be detected if no other contextual signal is present.
</Warning>

## How placeholders work

### Format

Each detected PII is replaced with a placeholder of the form `[TYPE_N]`:

```
"Jean"                     → [FIRST_NAME_1]
"Dupont"                   → [LAST_NAME_1]
"jean.dupont@acme.fr"      → [EMAIL_1]
"FR76 3000 6000 0112..."   → [IBAN_1]
"+33 6 12 34 56 78"        → [PHONE_NUMBER_1]
"12/03/1985"               → [DATE_OF_BIRTH_1]
```

### Multiple entities of the same type

Two different values of the same type get different numbers:

```
"Jean Dupont and Marie Martin" → "[FIRST_NAME_1] [LAST_NAME_1] and [FIRST_NAME_2] [LAST_NAME_2]"
```

### LLM system prompt tip

Add this instruction to prevent the LLM from modifying placeholders:

```
Preserve exactly all tokens in the format [TYPE_N] without any modification.
Do not translate, lowercase, or reformat tokens like [FIRST_NAME_1] or [IBAN_1].
```
