## Overview

There is no `/deanonymize` endpoint. Restoring original values happens **entirely in your code**, using the `entities` array returned by [`/extract`](/api-reference/extract).

This is by design — the mapping between placeholders and real values never leaves your infrastructure.

---

## How it works

The `/extract` endpoint returns:
- `text` — the anonymized text with placeholders (`[FIRST_NAME_1]`, `[IBAN_1]`…)
- `entities` — the list of detected entities with their original values, types, and positions

You build the mapping locally and do a string replace.

---

## Python

```python
import requests

# Call /extract
result = requests.post(
    "https://private-data.ai/api/v1/public/extract",
    headers={"X-API-Key": "your_api_key"},
    json={"text": "Send the report to Jean Dupont at jean@dupont.fr"}
).json()

# Build the local map
entity_map = {}
counters = {}
for entity in result["entities"]:
    t = entity["type"].upper()
    counters[t] = counters.get(t, 0) + 1
    entity_map[f"[{t}_{counters[t]}]"] = entity["value"]

# Simulate LLM response with placeholders intact
llm_response = "The report has been sent to [FIRST_NAME_1] [LAST_NAME_1] at [EMAIL_1]."

# Restore
restored = llm_response
for placeholder, original in entity_map.items():
    restored = restored.replace(placeholder, original)

print(restored)
# → "The report has been sent to Jean Dupont at jean@dupont.fr."
```

## JavaScript

```javascript
const response = await fetch("https://private-data.ai/api/v1/public/extract", {
  method: "POST",
  headers: { "X-API-Key": "your_api_key", "Content-Type": "application/json" },
  body: JSON.stringify({ text: "Send the report to Jean Dupont at jean@dupont.fr" })
});

const result = await response.json();

// Build local map
const counters = {};
const entityMap = {};
for (const entity of result.entities) {
  const t = entity.type.toUpperCase();
  counters[t] = (counters[t] || 0) + 1;
  entityMap[`[${t}_${counters[t]}]`] = entity.value;
}

// Restore in LLM response
const llmResponse = "The report has been sent to [FIRST_NAME_1] [LAST_NAME_1] at [EMAIL_1].";
let restored = llmResponse;
for (const [placeholder, original] of Object.entries(entityMap)) {
  restored = restored.replaceAll(placeholder, original);
}

console.log(restored);
// → "The report has been sent to Jean Dupont at jean@dupont.fr."
```

## Tips

<Warning>
  If the LLM modifies a placeholder (e.g. `[First_name_1]` instead of `[FIRST_NAME_1]`), the restore will silently fail for that entity. Add this instruction to your system prompt:

  *"Preserve exactly all tokens in the format [TYPE_N] without any modification."*
</Warning>

- **Store the map in memory** for the duration of a conversation — rebuild it fresh for each new user message.
- **Multiple entities of the same type** are numbered sequentially: `[FIRST_NAME_1]`, `[FIRST_NAME_2]`… Make sure your counter logic matches.
