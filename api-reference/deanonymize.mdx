---
title: "POST /deanonymize"
description: "Restaure les valeurs originales dans un texte contenant des tokens Data Private"
og:title: "Data Private API — /deanonymize"
---

## Endpoint

```
POST https://private-data.ai/api/v1/deanonymize
```

---

## Paramètres

### Headers

| Header | Valeur |
|---|---|
| `Authorization` | `Bearer dp_...` (requis) |
| `Content-Type` | `application/json` (requis) |

### Body

| Paramètre | Type | Requis | Description |
|---|---|---|---|
| `text` | `string` | ✅ | Texte contenant des tokens à réidentifier. Typiquement la réponse du LLM. |
| `tokens` | `object` | Non* | Table de mapping `token → valeur originale`. Requis si `session_id` n'est pas fourni. |
| `session_id` | `string` | Non* | Identifiant de session. Si fourni, Data Private récupère automatiquement le dictionnaire de mapping — pas besoin de passer `tokens`. |
| `fail_on_missing` | `boolean` | Non | Si `true`, retourne une erreur si un token du texte est absent du mapping. Défaut : `false`. |

<Note>
  `tokens` ou `session_id` — l'un des deux est requis. Si les deux sont fournis, `session_id` est prioritaire.
</Note>

---

## Exemples

<CodeGroup>
```python Python
import dataprivate

dp = dataprivate.Client(api_key="dp_...")

# Sans session — passe la table de tokens explicitement
tokens = {
    "PERSON_1": "Jean Dupont",
    "IBAN_1": "FR76 3000 6000 0112 3456 7890"
}
llm_response = "Le virement de [PERSON_1] vers [IBAN_1] a été validé."

result = dp.deanonymize(text=llm_response, tokens=tokens)
print(result)
# → "Le virement de Jean Dupont vers FR76 3000 6000 0112 3456 7890 a été validé."

# Avec session — Data Private gère le dictionnaire
result = dp.deanonymize(
    text=llm_response,
    session_id="ses_abc123"
)
print(result)
```

```javascript JavaScript
import DataPrivate from 'dataprivate';

const dp = new DataPrivate({ apiKey: 'dp_...' });

// Sans session
const tokens = {
  PERSON_1: 'Jean Dupont',
  IBAN_1: 'FR76 3000 6000 0112 3456 7890'
};
const llmResponse = 'Le virement de [PERSON_1] vers [IBAN_1] a été validé.';

const result = await dp.deanonymize({ text: llmResponse, tokens });
console.log(result);
// → 'Le virement de Jean Dupont vers FR76 3000 6000 0112 3456 7890 a été validé.'

// Avec session
const result2 = await dp.deanonymize({
  text: llmResponse,
  sessionId: 'ses_abc123'
});
```

```bash curl
# Sans session
curl -X POST https://private-data.ai/api/v1/deanonymize \
  -H "Authorization: Bearer dp_..." \
  -H "Content-Type: application/json" \
  -d '{
    "text": "Le virement de [PERSON_1] vers [IBAN_1] a été validé.",
    "tokens": {
      "PERSON_1": "Jean Dupont",
      "IBAN_1": "FR76 3000 6000 0112 3456 7890"
    }
  }'

# Avec session
curl -X POST https://private-data.ai/api/v1/deanonymize \
  -H "Authorization: Bearer dp_..." \
  -H "Content-Type: application/json" \
  -d '{
    "text": "Le virement de [PERSON_1] vers [IBAN_1] a été validé.",
    "session_id": "ses_abc123"
  }'
```
</CodeGroup>

## Réponse

```json
{
  "text": "Le virement de Jean Dupont vers FR76 3000 6000 0112 3456 7890 a été validé.",
  "replacements": [
    { "token": "PERSON_1", "value": "Jean Dupont", "start": 15, "end": 26 },
    { "token": "IBAN_1", "value": "FR76 3000 6000 0112 3456 7890", "start": 33, "end": 61 }
  ]
}
```

## Codes d'erreur

| Code HTTP | Code erreur | Description |
|---|---|---|
| `400` | `invalid_request` | Ni `tokens` ni `session_id` fourni |
| `400` | `missing_tokens` | Token présent dans le texte mais absent du mapping (si `fail_on_missing: true`) |
| `401` | `unauthorized` | Clé API manquante ou invalide |
| `404` | `session_not_found` | `session_id` inconnu ou expiré |
| `429` | `rate_limit_exceeded` | Limite de requêtes atteinte |
| `500` | `internal_error` | Erreur serveur |

---

## Piège courant

<Warning>
  Si le LLM reformule un token (ex : `[Person 1]` au lieu de `[PERSON_1]`), la réidentification échoue silencieusement sur ce token. Ajoute cette instruction dans ton system prompt :

  *"Preserve exactly all tokens in the format [TYPE_N] without any modification."*
</Warning>
